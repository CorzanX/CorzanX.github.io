Deep Insights

  1. Time Complexity Growth (O(n³))
    The execution time increases rapidly as the matrix size grows. This is expected because matrix multiplication has a time complexity of O(n³). Doubling the matrix size results in approximately eight times more computations.
  2. Compiled vs Interpreted Execution
    The C implementation is significantly faster because it is compiled into machine code before execution. In contrast, Python is interpreted, introducing overhead at runtime for each operation.
  3. Memory Access and Cache Efficiency
    C provides better control over memory layout, leading to more efficient cache usage. Python’s high-level data structures (lists of lists) introduce additional overhead and less predictable memory access patterns.
  4. Loop Overhead
    Python suffers from higher loop overhead due to dynamic typing and interpreter checks in each iteration, whereas C executes loops much more efficiently.
  5. Scalability Observation
    As the matrix size increases, the performance gap between C and Python becomes more significant, indicating that C is more suitable for computation-intensive tasks.
  6. Potential Optimization Opportunities
    • Using optimized libraries such as NumPy in Python can drastically improve performance.
    • Applying advanced algorithms can reduce time complexity.

Conclusion

In this project, we implemented matrix multiplication using both C and Python, and explored the differences between compiled and interpreted languages.

We also gained practical experience with:

  • Unix/Linux command line operations
  • Markdown documentation
  • Program compilation and execution

The results show that C provides better performance, while Python offers easier implementation and readability.
