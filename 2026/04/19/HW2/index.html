The website is a static personal blog generated by Hexo. It can be accessed through the GitHub Pages URL above after deployment. The first assignment report has already been added as a blog post and is integrated into the same website.

2. Objective

The goal of this assignment is to set up and deploy a static personal blog website, manage the project with Git, integrate the previous Markdown report, and document the full process.

This project satisfies the requirements in the following ways:

  1. The blog is built as a static website with Hexo.
  2. The site is deployed to GitHub Pages.
  3. The previous assignment report, HW1.md, is included as a post.
  4. The project is managed with Git commits that represent different stages of development.
  5. This Markdown report documents the setup, deployment, Git process, and tool choices.

3. Project Structure

source/_posts/HW2.md This assignment report
public/ Generated static HTML files
.deploy_git/ Temporary deployment Git directory generated by Hexo deployer

4. Tool and Framework Choice

I chose Hexo as the static site generator because this blog had already been created with Hexo before, and the existing folder structure was still usable. Reusing the existing Hexo project made the process efficient and avoided rebuilding the website from zero.

Reasons for using Hexo:

  • It converts Markdown posts into static HTML pages.
  • It is suitable for personal blogs and assignment reports.
  • It has a clear project structure.
  • It supports themes and plugins.
  • It can deploy directly to GitHub Pages with hexo-deployer-git.

I used the Butterfly theme because it was already configured in this project and provides a complete blog layout, including post pages, category pages, tag pages, and local search.

10. Reflection

This assignment helped me reconnect with a Hexo blog project that had not been actively maintained for several years. Instead of discarding the old project, I inspected its structure, confirmed the deployment configuration, reused the existing theme, and added new assignment content.

The most useful part of the process was understanding the relationship between Markdown source files and generated static pages:

  • Markdown files in source/_posts/ are the source content.
  • Hexo converts them into HTML pages.
  • GitHub Pages serves the final static website.
  • Git records the development history.

This workflow is simple but practical. It is suitable for course reports, technical notes, and personal documentation.
